# 🧠 RAG知识库系统 - 项目详细介绍

## 项目概述

这是一个基于检索增强生成（Retrieval-Augmented Generation, RAG）技术构建的智能知识库问答系统。项目旨在帮助用户构建基于私有文档的精准问答能力，能够处理PDF、Word、Excel、Markdown、文本等多种格式的文档，并通过先进的AI技术实现智能检索和问答。

## 核心特性

### 1. 多格式文档支持
- **PDF文档**：使用PyMuPDF进行高效解析
- **Office文档**：支持Word、Excel等格式
- **文本文件**：Markdown、TXT、代码文件等
- **网页内容**：支持网页抓取和解析

### 2. 智能检索策略
- **语义检索**：基于向量相似度的语义匹配
- **混合检索**：结合语义和关键词检索的优势
- **多查询扩展**：通过LLM生成多个查询角度，提高召回率

### 3. 灵活的分块策略
- **递归分块**：按层级分隔符进行分块
- **句子分块**：保持语义完整性
- **段落分块**：保留上下文信息

### 4. 强大的LLM后端支持
- **Claude API**：云端高性能模型
- **Ollama**：本地模型部署
- **可扩展架构**：易于集成新的LLM后端

### 5. 安全保障
- **敏感信息保护**：密钥管理、环境变量配置
- **路径遍历防护**：防止目录穿越攻击
- **SQL注入防护**：数据库操作安全验证

## 技术栈

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| LLM后端 | Claude API / Ollama | 支持云端和本地模型 |
| 向量数据库 | ChromaDB | 轻量级向量存储 |
| 嵌入模型 | BAAI/bge-small-zh-v1.5 | 中文小型嵌入模型，~100MB |
| 关系数据库 | MySQL | 元数据和会话记忆存储 |
| Web框架 | FastAPI | RESTful API服务 |
| 命令行 | Click | CLI交互 |

## 快速开始

### 环境要求
- Python 3.10+
- MySQL 8.0+（可选，用于元数据存储）

### 安装步骤
```bash
# 克隆项目
git clone <repository-url>
cd Rag

# 创建虚拟环境
python -m venv .venv
source .venv/bin/activate  # Linux/Mac
# .venv\Scripts\activate   # Windows

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp .env.example .env
# 编辑 .env，填入 API Key 等敏感信息
```

### 基本使用
```bash
# 导入文档
python run.py add docs/*.md

# 提问（默认语义检索）
python run.py ask "什么是梯度下降？"

# 混合检索（适合专业术语查询）
python run.py ask "什么是梯度下降？" --hybrid

# 多查询扩展（适合复杂问题）
python run.py ask "RAG系统的基本原理是什么？" --multi-query

# 查看知识库文档列表
python run.py list

# 启动API服务
python run_api.py
```

## 项目优势

1. **准确性优先**：只基于知识库内容回答，杜绝幻觉
2. **本地化部署**：数据不出本地，隐私安全
3. **检索质量高**：混合检索+多查询扩展，最大化召回率
4. **易于扩展**：模块化设计，便于定制功能
5. **性能优化**：支持GPU加速嵌入计算，提升处理速度
6. **安全可靠**：完善的权限控制和安全机制
