# 📚 RAG分块使用指南

## 快速开始

### 1. 了解分块策略

RAG系统提供三种分块策略，可根据文档类型选择：

- **recursive** (默认) - 递归分块，适合通用文档
- **sentence** - 句子分块，保持句子完整性，适合技术文档
- **paragraph** - 段落分块，保持段落完整性，适合结构化内容

### 2. 修改配置

在 `config.yaml` 中调整分块参数：

```yaml
chunking:
  size: 512        # 每块最大字符数 (默认512)
  overlap: 64      # 块间重叠字符数 (默认64)  
  strategy: sentence  # 分块策略 (recursive|sentence|paragraph)
```

### 3. 导入文档

```bash
# 导入文档，系统会自动使用配置的分块策略
python run.py add your_document.pdf

# 可以通过API指定不同的分块参数（如果系统支持）
```

## 分块参数调优指南

### 文档类型推荐配置

| 文档类型 | 推荐策略 | 块大小 | 重叠 | 说明 |
|----------|----------|--------|------|------|
| 技术文档 | sentence | 384 | 48 | 保持概念完整性 |
| 学术论文 | sentence | 512 | 64 | 保持公式和解释关联 |
| 小说文章 | paragraph | 768 | 96 | 保持段落连贯性 |
| 网页内容 | recursive | 512 | 64 | 适应多样化内容 |
| 代码文档 | recursive | 512 | 32 | 包含完整代码段 |

### 参数含义

- **chunk_size**: 控制每个文本块的最大字符数
  - 较小值（256-384）：更精确的匹配，但可能丢失上下文
  - 较大值（768-1024）：保留更多上下文，但可能包含无关信息

- **overlap**: 控制相邻文本块之间的重叠字符数
  - 确保查询时不会遗漏跨越块边界的上下文
  - 通常设为chunk_size的10%-15%

- **strategy**: 选择最适合文档结构的分块策略

## 实际使用技巧

### 1. 根据用途调整

**用于问答系统**：较小的块大小（384-512）和句子分块策略
**用于文档摘要**：较大的块大小（768-1024）和段落分块策略
**用于代码检索**：中等块大小（512）和递归分块策略

### 2. 检查分块效果

导入文档后，可以用以下命令查看文档信息：

```bash
# 查看已导入的文档列表
python run.py list

# 查看系统状态
python run.py info
```

### 3. 性能考虑

- 块大小影响检索精度和速度：较小的块提供更精确的结果，但需要检索更多候选
- 重叠增加存储和计算开销，但提高上下文连续性
- 选择与您的硬件资源匹配的配置

## 常见问题

Q: 导入文档后检索不到相关信息？
A: 尝试调整chunk_size，可能文档块太大或太小

Q: 检索结果缺少上下文？
A: 增加重叠大小或尝试段落分块策略

Q: 检索速度很慢？
A: 考虑减小chunk_size或使用句子分块策略减少检索的块数量

Q: 答案缺乏连贯性？
A: 增加chunk_size或使用段落分块策略保持更多上下文

通过调整这些参数，您可以优化RAG系统在您的特定应用场景下的表现。
