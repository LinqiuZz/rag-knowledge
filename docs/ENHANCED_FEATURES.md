# 增强的RAG功能说明

本文档介绍了项目中新增的增强功能，旨在提高问答系统的准确性和召回率。

## 1. 混合检索（Hybrid Search）

### 功能说明
混合检索结合了语义检索和关键词检索的优势：
- **语义检索**：理解查询的语义含义，找到语义相似的文档
- **关键词检索**：精确匹配关键词，确保专业术语的准确性

### 使用方法
```bash
# 使用混合检索
python run.py ask "你的问题" --hybrid

# 示例
python run.py ask "什么是梯度下降？" --hybrid
```

### 技术实现
- 语义权重：0.7（默认）
- 关键词权重：0.3（默认）
- 使用倒排索引进行关键词匹配
- 使用向量相似度进行语义匹配

## 2. 多查询检索（Multi-Query Retrieval）

### 功能说明
多查询检索通过LLM生成多个相关查询来提高检索召回率：
- 自动生成多个语义相似的查询
- 从不同角度检索相关文档
- 去重后返回最相关的文档

### 使用方法
```bash
# 使用多查询检索
python run.py ask "你的问题" --multi-query

# 示例
python run.py ask "RAG系统的基本原理是什么？" --multi-query
```

### 技术实现
- 使用LLM生成3个相关查询（可配置）
- 对每个查询进行语义检索
- 按来源去重，保留最相关的结果

## 3. 增强的分块策略

### 功能说明
支持多种分块策略，适应不同类型的文档：
- **递归字符切分**：按固定长度切分，保持重叠
- **句子级切分**：保持句子完整性（默认）
- **段落级切分**：保持段落完整性

### 配置方法
在 `config.yaml` 中配置：
```yaml
chunking:
  size: 512          # 每块目标字符数
  overlap: 64        # 块间重叠字符数
  strategy: sentence  # 分块策略: recursive | sentence | paragraph
```

### 策略选择建议
- **技术文档**：使用 `sentence` 策略，保持技术概念的完整性
- **长篇文章**：使用 `paragraph` 策略，保持段落逻辑
- **代码文件**：使用 `recursive` 策略，按函数/类切分

## 4. 优化的提示词

### 功能说明
优化后的提示词模板能够生成更专业、更结构化的回答：
- 强调准确性优先
- 要求标注信息来源
- 鼓励结构化回答
- 保持专业术语准确性

### 提示词内容
```
你是一个专业的技术问答助手。请严格按照以下规则回答：

1. **准确性优先**：只基于提供的参考资料回答，绝不编造信息
2. **来源标注**：每个关键信息点必须标注来源 [来源N]
3. **结构化回答**：使用标题、列表、代码块等结构化格式
4. **完整性检查**：如果参考资料不足，明确说明"根据现有资料无法完整回答"
5. **专业术语**：保持技术术语的准确性，必要时给出解释

回答风格：专业、准确、有条理、易于理解
```

## 5. 评估指标

### 检索质量指标
- **Precision@K**：前K个检索结果中相关文档的比例
- **Recall@K**：相关文档被检索到的比例
- **MRR（Mean Reciprocal Rank）**：第一个相关文档排名的倒数
- **NDCG（Normalized Discounted Cumulative Gain）**：考虑排名位置的相关性评分

### 生成质量指标
- **忠实度（Faithfulness）**：回答是否忠实于检索到的上下文
- **相关性（Relevance）**：回答是否与问题相关
- **完整性（Completeness）**：回答是否完整地回答了问题

## 6. 最佳实践

### 知识库管理
1. **文档质量**：确保知识库中的文档准确、权威、及时
2. **文档覆盖**：覆盖用户可能询问的各个主题
3. **定期更新**：定期更新知识库，确保信息时效性

### 检索策略选择
1. **简单查询**：使用纯语义检索
2. **专业术语查询**：使用混合检索
3. **复杂查询**：使用多查询检索
4. **长文档查询**：调整分块策略和参数

### 参数调优
1. **top_k**：根据查询复杂度调整（默认5）
2. **chunk_size**：根据文档类型调整（默认512）
3. **overlap**：根据文档结构调整（默认64）

## 8. 常见问题

### Q1: 混合检索和多查询检索应该用哪个？
- **混合检索**：适合包含专业术语的查询
- **多查询检索**：适合复杂、多角度的查询
- 可以结合使用：先使用多查询扩展，再使用混合检索

### Q2: 如何调整检索参数？
在 `config.yaml` 中调整：
```yaml
chunking:
  size: 512        # 增大可提高上下文连贯性
  overlap: 64      # 增大可减少信息断裂
  strategy: sentence
```

### Q3: 如何提高回答质量？
1. 确保知识库有相关文档
2. 使用混合检索或混合检索
3. 调整top_k参数（增大可获得更多上下文）
4. 优化提示词模板
