# 机器学习基础概念

## 1. 什么是机器学习

机器学习（Machine Learning, ML）是人工智能的一个子领域，它使计算机系统能够通过经验自动改进，而无需显式编程。机器学习算法通过从数据中学习模式和规律，来进行预测或决策。

### 1.1 机器学习的定义

Arthur Samuel（1959）：机器学习是"赋予计算机无需显式编程就能学习的能力的研究领域"。

Tom Mitchell（1997）：对于某类任务 T 和性能度量 P，如果计算机程序在 T 上以 P 衡量的性能随着经验 E 而自我完善，则称该程序从经验 E 中学习。

### 1.2 机器学习的发展历史

- **1943年**：McCulloch 和 Pitts 提出人工神经元模型
- **1950年**：Alan Turing 发表《计算机器与智能》，提出图灵测试
- **1957年**：Frank Rosenblatt 发明感知机（Perceptron）
- **1969年**：Minsky 和 Papert 指出感知机的局限性，AI 进入第一次寒冬
- **1986年**：Rumelhart 等人提出反向传播算法（Backpropagation）
- **1997年**：SVM（支持向量机）开始流行
- **2006年**：Hinton 提出深度学习概念
- **2012年**：AlexNet 在 ImageNet 竞赛中大幅领先，深度学习革命开始
- **2017年**：Transformer 架构问世（"Attention is All You Need"）
- **2022年**：ChatGPT 发布，大语言模型时代来临

## 2. 机器学习的分类

### 2.1 监督学习（Supervised Learning）

监督学习是机器学习中最常见的范式。算法从带有标签的训练数据中学习输入到输出的映射关系。

**核心思想**：给定输入 X 和对应标签 Y，学习函数 f: X → Y

**常见算法**：
- **线性回归（Linear Regression）**：用于连续值预测
- **逻辑回归（Logistic Regression）**：用于二分类问题
- **决策树（Decision Tree）**：基于树结构进行决策
- **随机森林（Random Forest）**：多棵决策树的集成
- **支持向量机（SVM）**：寻找最优分隔超平面
- **K 近邻（KNN）**：基于最近邻投票
- **朴素贝叶斯（Naive Bayes）**：基于贝叶斯定理的概率分类

**应用场景**：
- 图像分类
- 垃圾邮件检测
- 房价预测
- 医疗诊断

### 2.2 无监督学习（Unsupervised Learning）

无监督学习处理没有标签的数据，目标是发现数据中的隐藏结构和模式。

**常见算法**：
- **K-Means 聚类**：将数据分成 K 个簇
- **DBSCAN**：基于密度的聚类算法
- **层次聚类**：构建聚类层次结构
- **PCA（主成分分析）**：降维技术
- **t-SNE**：高维数据可视化
- **自编码器（Autoencoder）**：学习数据的压缩表示

**应用场景**：
- 客户分群
- 异常检测
- 数据降维
- 特征学习

### 2.3 强化学习（Reinforcement Learning）

强化学习中，智能体通过与环境交互来学习最优策略，通过试错获得最大累积奖励。

**核心概念**：
- **智能体（Agent）**：学习和决策的主体
- **环境（Environment）**：智能体交互的外部世界
- **状态（State）**：环境的当前情况
- **动作（Action）**：智能体可执行的操作
- **奖励（Reward）**：环境对动作的反馈
- **策略（Policy）**：从状态到动作的映射

**常见算法**：
- Q-Learning
- Deep Q-Network (DQN)
- Policy Gradient
- Actor-Critic (A2C/A3C)
- Proximal Policy Optimization (PPO)

**应用场景**：
- 游戏 AI（AlphaGo）
- 机器人控制
- 自动驾驶
- 推荐系统优化

### 2.4 半监督学习（Semi-Supervised Learning）

结合少量有标签数据和大量无标签数据进行学习，是监督学习和无监督学习之间的折中方案。

### 2.5 自监督学习（Self-Supervised Learning）

从无标签数据中自动生成监督信号，是当前大语言模型预训练的主要方式。例如 BERT 的掩码语言模型任务和 GPT 的下一个词预测任务。

## 3. 机器学习的核心概念

### 3.1 特征工程（Feature Engineering）

特征工程是将原始数据转换为更适合机器学习模型使用的特征的过程。

**常用技术**：
- **特征缩放**：标准化（Z-score）、归一化（Min-Max）
- **特征编码**：独热编码（One-Hot）、标签编码（Label Encoding）
- **特征选择**：过滤法、包装法、嵌入法
- **特征提取**：PCA、LDA、TF-IDF
- **缺失值处理**：均值填充、中位数填充、插值法

### 3.2 模型评估指标

**分类指标**：
- **准确率（Accuracy）**：正确预测数 / 总预测数
- **精确率（Precision）**：真正例 / (真正例 + 假正例)
- **召回率（Recall）**：真正例 / (真正例 + 假负例)
- **F1 分数**：精确率和召回率的调和平均
- **ROC-AUC**：受试者工作特征曲线下面积

**回归指标**：
- **MSE（均方误差）**：预测值与真实值差的平方的平均
- **RMSE（均方根误差）**：MSE 的平方根
- **MAE（平均绝对误差）**：预测值与真实值差的绝对值的平均
- **R² 分数**：决定系数，表示模型解释数据方差的比例

### 3.3 偏差-方差权衡（Bias-Variance Tradeoff）

- **偏差（Bias）**：模型预测值与真实值之间的系统性偏离，高偏差意味着欠拟合
- **方差（Variance）**：模型对训练数据变化的敏感程度，高方差意味着过拟合
- **目标**：找到偏差和方差之间的最佳平衡点

### 3.4 正则化（Regularization）

正则化是防止模型过拟合的技术。

- **L1 正则化（Lasso）**：添加权重绝对值之和的惩罚项，可产生稀疏解
- **L2 正则化（Ridge）**：添加权重平方和的惩罚项，使权重趋于较小值
- **Dropout**：神经网络中随机丢弃部分神经元
- **早停法（Early Stopping）**：验证集性能不再提升时停止训练

### 3.5 交叉验证（Cross-Validation）

交叉验证是一种模型评估方法，最常用的是 K 折交叉验证：

1. 将数据集分成 K 份
2. 每次用 K-1 份训练，1 份验证
3. 重复 K 次，取平均性能

常用 K=5 或 K=10。

## 4. 深度学习基础

### 4.1 神经网络基础

人工神经网络受生物神经系统启发，由相互连接的神经元层组成。

**基本组件**：
- **输入层**：接收原始数据
- **隐藏层**：执行特征变换
- **输出层**：产生最终预测
- **激活函数**：引入非线性（ReLU、Sigmoid、Tanh、Softmax）

### 4.2 常见网络架构

**卷积神经网络（CNN）**：
- 专门处理网格结构数据（如图像）
- 核心操作：卷积、池化、全连接
- 经典模型：LeNet、AlexNet、VGG、ResNet、EfficientNet

**循环神经网络（RNN）**：
- 处理序列数据（如文本、时间序列）
- 变体：LSTM（长短期记忆）、GRU（门控循环单元）
- 问题：梯度消失/爆炸

**Transformer**：
- 基于自注意力机制（Self-Attention）
- 并行计算，效率高于 RNN
- 是 BERT、GPT 等大模型的基础架构

**生成对抗网络（GAN）**：
- 由生成器和判别器组成
- 生成器生成假数据，判别器区分真假
- 应用：图像生成、风格迁移、数据增强

### 4.3 训练过程

1. **前向传播**：输入数据通过网络产生输出
2. **计算损失**：比较输出与真实标签的差异
3. **反向传播**：计算损失对每个参数的梯度
4. **参数更新**：使用优化算法更新参数

**常见优化器**：
- SGD（随机梯度下降）
- SGD with Momentum
- AdaGrad
- RMSProp
- Adam（最常用）

**学习率调度**：
- 学习率衰减
- 余弦退火
- Warmup

## 5. 机器学习工作流程

一个完整的机器学习项目通常包含以下步骤：

### 5.1 问题定义
- 明确业务目标
- 确定机器学习任务类型
- 定义评估指标

### 5.2 数据收集
- 确定数据来源
- 数据采集和存储
- 数据标注（如需要）

### 5.3 数据探索（EDA）
- 数据统计描述
- 可视化分析
- 相关性分析
- 发现数据质量问题

### 5.4 数据预处理
- 缺失值处理
- 异常值处理
- 特征编码
- 特征缩放
- 数据集划分（训练集/验证集/测试集）

### 5.5 模型选择与训练
- 选择候选模型
- 超参数调优（网格搜索、随机搜索、贝叶斯优化）
- 模型训练

### 5.6 模型评估
- 在测试集上评估
- 分析错误案例
- 对比基线模型

### 5.7 模型部署
- 模型序列化（pickle、ONNX、TorchScript）
- API 服务化
- 监控和维护

## 7. 机器学习最佳实践

1. **从简单模型开始**：先用基线模型（如逻辑回归）建立基准
2. **理解数据比选模型更重要**：数据质量决定上限，模型只是逼近
3. **避免数据泄漏**：确保训练时不用到测试集信息
4. **关注可解释性**：复杂模型不一定优于简单可解释的模型
5. **持续监控**：模型上线后需要监控性能衰减（数据漂移）
6. **版本控制**：对代码、数据和模型都进行版本管理
7. **文档记录**：记录实验过程、参数和结果

## 8. 机器学习的挑战与伦理

### 8.1 常见挑战
- 数据不足或数据质量差
- 类别不平衡
- 维度灾难
- 计算资源限制
- 模型可解释性不足

### 8.2 伦理问题
- **偏见与公平性**：模型可能继承训练数据中的偏见
- **隐私保护**：用户数据的收集和使用
- **透明度**：模型决策过程的可解释性
- **责任归属**：AI 决策出错时的责任问题
