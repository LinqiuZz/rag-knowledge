# 系统架构

## 整体设计

本系统是一个 **本地优先** 的个人知识库，核心思路是 **摄取 → 向量化 → 检索 → 生成**。

所有文档数据存储在本地磁盘，嵌入计算在本地 CPU/GPU 完成，唯一需要联网的环节是使用 Claude API 时的 LLM 调用。

                        │        用户交互层 (CLI)       │

              ┌────────────────────────┼────────────────────────┐

     │    摄取管道       │    │    查询引擎          │   │    管理命令      │
     │ PDF → 分块 → 嵌入 │    │ 搜索 / 摘要 / RAG   │   │ list / remove   │

              │                        │

## 数据流

### 文档摄取流程

```
PDF 文件 / URL
    │
    ▼
┌─────────────┐     ┌──────────────┐     ┌───────────────┐
│  内容提取    │────▶│  文本分块     │────▶│  本地嵌入      │
│ PyMuPDF /   │     │ 递归字符切分  │     │ bge-small-zh  │
│ trafilatura │     │ 512字/块      │     │ sentence-trans│
└─────────────┘     └──────────────┘     └───────┬───────┘
                                                  │
                    ┌──────────────┐     ┌────────▼───────┐
                    │  MySQL       │◀────│  ChromaDB      │
                    │  元数据记录   │     │  向量存储       │
                    └──────────────┘     └────────────────┘
```

### RAG 问答流程

```
用户提问
    │
    ▼
┌─────────────┐     ┌──────────────┐     ┌───────────────┐
│  问题嵌入    │────▶│  向量检索     │────▶│  构建上下文    │
│ 本地嵌入模型 │     │  Top-K 相似块 │     │ 拼接来源引用   │
└─────────────┘     └──────────────┘     └───────┬───────┘
                                                  │
                    ┌──────────────┐     ┌────────▼───────┐
                    │  返回结果     │◀────│  LLM 生成回答  │
                    │  answer +    │     │ Claude / Ollama│
                    │  sources     │     └────────────────┘
                    └──────────────┘
```

## 模块依赖关系

```
src/
├── config.py              ← 被所有模块依赖，读取 config.yaml
├── cli.py                 ← 顶层入口，组装各模块
│
├── ingest/                ← 文档摄取
│   ├── pdf.py             ← 依赖 fitz (PyMuPDF)
│   ├── webpage.py         ← 依赖 trafilatura
│   ├── chunking.py        ← 纯 Python，无外部依赖
│   └── pipeline.py        ← 组装 pdf/webpage + chunking + store
│
├── store/                 ← 存储层
│   ├── embedding.py       ← 依赖 sentence-transformers
│   ├── vector.py          ← 依赖 chromadb
│   └── metadata.py        ← 依赖 mysql-connector-python
│
├── llm/                   ← LLM 后端
│   ├── base.py            ← 抽象接口 + 工厂函数
│   ├── claude.py          ← 依赖 anthropic
│   └── ollama.py          ← 依赖 ollama
│
└── query/                 ← 查询引擎
    ├── search.py          ← 依赖 store/vector + store/embedding
    ├── summarize.py       ← 依赖 search + llm
    └── rag.py             ← 依赖 search + llm
```

## 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 向量数据库 | ChromaDB | 嵌入式、零配置、纯本地、支持 cosine 距离 |
| 嵌入模型 | BAAI/bge-small-zh-v1.5 | 中文优化、体积小 (~100MB)、纯本地运行 |
| 嵌入运行时 | sentence-transformers + CUDA | GPU 加速推理，RTX 3070 Ti 上毫秒级嵌入 |
| PDF 解析 | PyMuPDF (fitz) | 速度最快、支持加密 PDF、API 简洁 |
| 网页提取 | trafilatura | 比 BeautifulSoup 更擅长正文提取、自带去噪 |
| 分块策略 | 递归字符切分 | 按语义边界切分 (段落→句子→字符)，比固定窗口好 |
| 元数据存储 | MySQL  | 支持并发、结构化查询、生产级 |
| LLM 抽象 | BaseLLM + 工厂 | 一行配置切换 Claude/Ollama，易于扩展 |
| 配置格式 | YAML | 可读性好、支持注释 |
| CLI 框架 | Click + Rich | Click 命令解析成熟，Rich 终端输出美观 |
